Bonus: AI Prompt Templates

Getting Started with AI in Discovery

// Understanding AI Capabilities Prompt
const discoveryPrompt = `
You are a product discovery expert. Help me understand:

1. What AI can do well in product research
2. What AI cannot replace (human judgment, empathy)
3. How to use AI as a thinking partner, not replacement

Provide 3 specific examples of AI-human collaboration
in user research.
`;

// Pro tip: Start broad, then iterate based on results
Share this template with students. Emphasize that AI is a collaborator, not a replacement. The best results come from combining AI capabilities with human intuition.

AI-Assisted Research Planning

// Generating Research Questions
const researchPrompt = `
I'm researching [your product area here].

Context:
- Target users: [describe users]
- Current pain points: [known issues]
- Research goals: [what you want to learn]

Generate 10 open-ended interview questions that will help me understand:
1. User motivations and goals
2. Current workflow and pain points
3. Desired outcomes and success criteria
4. Context and environment of use

Format as: Category | Question | Follow-up probes
`;
Encourage students to provide rich context. The more context AI has, the better the questions. Always review and refine AI-generated questions before using them.

Analyzing User Interviews with AI

// Interview Analysis Template
const analysisPrompt = `
Analyze this user interview transcript and identify:

üìä Key Insights:
1. Pain points mentioned (with severity indicators)
2. Unmet needs or desires
3. Workarounds the user has created
4. Jobs-to-be-done
5. Emotional reactions to current solutions

üìù Evidence:
- Include direct quotes for each insight

Format output as structured JSON for easy analysis.

This is a powerful template for post-interview analysis. Emphasize the importance of keeping original quotes and context. AI can find patterns, but humans validate meaning.

Synthesis: From Data to Insights

// Cross-Interview Synthesis
const synthesisPrompt = `
Based on these insights from 5 user interviews:

[Paste 8-10 key quotes or observations from different users]

Identify:
1. **Common themes** across users (3-5 themes)
2. **Behavioral patterns** - what users actually do vs say
3. **User segments** - are there distinct groups?
4. **Priority insights** - what should drive product decisions?

For each theme:
- Theme name
- Supporting evidence (quotes from multiple users)
- Implications for product development
- Confidence level (high/medium/low)
`;
The synthesis stage is where AI really shines - finding patterns across multiple conversations. Always validate AI-identified themes with your own review of the raw data.

Advanced: AI-Assisted Persona Creation

// Persona Generation from Research
const personaPrompt = `
Based on this user research data:
- 10 user interviews (summaries below)
- Identified themes: [list your themes]
- Behavioral patterns: [list patterns]

Create 2-3 distinct user personas that represent
our user base.

For each persona include:
1. Name and role
2. Goals and motivations
3. Pain points and frustrations
4. Current workflow
5. Technology comfort level
6. Quote that captures their mindset

Base personas on ACTUAL research data, not assumptions.
Include evidence from interviews to support each trait.
`;
Personas should emerge from research, not imagination. Use AI to synthesize patterns, but always ground personas in real user quotes and behaviors.
